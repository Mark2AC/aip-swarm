# VISION — AIP Swarm 계획과 비전

> 이 프로젝트가 **무엇을·왜** 만드는지. 단계별 실행 계획은 [`ROADMAP.md`](ROADMAP.md),
> 현재 시스템 구조는 [`ARCHITECTURE.md`](ARCHITECTURE.md), 설치·운용은 [`USAGE.md`](USAGE.md).

---

## 1. 핵심 목표

**동등한 자율 이동체들이 중앙 지휘 없이 협력하는 분산 군집 시스템.**

각 차량은:
- 독립적으로 자기 위치를 파악하고
- 독립적으로 임무를 수행할 수 있으며
- 다른 차량과 협력할 때 더 나은 성과를 낸다

**단일 차량 고장이 전체 군집의 임무를 중단시키지 않는다.**

## 2. 프로젝트 개요

실내/창고 환경에서 다중 AGV가 **SLAM 기반 자율주행 + 순찰 + 열화상 이상 감지**를 수행하는
군집 로봇 시스템. 웹 대시보드로 관제한다. 중앙 제어 AI(Fleet Brain)가 커버리지 배치·이상 대응·
상태 관리를 **제안(advisory)** 하도록 설계돼 있으며, 현재는 *설계 단계*다([`CENTRAL_AI.md`](CENTRAL_AI.md)).

- **차량 3대**: `aip1`(메인 AGV, 완전 사양) · `aip2`(TurtleBot3 Burger) · `aip3`(자작 차량)
- **중앙 PC**: 관제 대시보드 + 상태 감시(supervisor/watchdog) [+ Fleet Brain(계획)]
- **통신**: ROS2 Humble + FastDDS, 전용 Wi-Fi AP `AIP_FLEET`(192.168.0.0/24)

## 3. 현재와 목표의 괴리 — 예산 제약

이상적으로는 모든 차량이 동등한 사양이어야 하나, **메인 AGV급을 여러 대 제작할 예산이 없어**
현재는 임시 계층 구조가 존재한다.

```
목표 (동질 군집)              현재 (예산 제약 군집)
──────────────────────        ──────────────────────────────
[차량 A] ←→ [차량 B]          [메인 AGV(aip1)] ←→ [보조 차량 × 2]
    ↕            ↕            (LiDAR·SLAM·독립측위)  (최소 사양·중앙 의존)
[차량 C] ←→ [차량 D]
```

## 4. 용어 정의

| 코드 내 용어 | 실제 의미 | 임시 여부 |
|---|---|---|
| `aip1` | 완전 사양 피어 (1세대 메인 AGV) | 임시 — 향후 모든 차량이 동급 |
| `aip2`/`aip3` | 예산 제약 피어 (1세대) | 임시 — 사양 향상 시 동급 전환 |
| `supervisor` | 군집 상태 모니터 (중앙 PC) | 임시 — 장기적으로 각 차량에 분산 |
| `coordinator` | 편대 추종 제어기 (중앙 PC) | 임시 — 장기적으로 각 차량 탑재 |
| `Fleet Brain` | 중앙 제어 AI (advisory, 설계 단계) | 유지 — 제안만, 차량 직접 제어 안 함 |

## 5. 설계 원칙

**1. 코드는 목표 아키텍처를 향해 쓴다.** 현재 중앙 PC 기능(coordinator, supervisor)은 나중에 각
차량으로 이동할 것을 전제로 작성한다. 하드코딩된 차량 목록이 아니라 **하트비트 기반 자동 발견**을 지향.

**2. 네임스페이스는 차량 정체성이지 역할이 아니다.** `/aip1/cmd_vel`은 "메인이라서"가 아니라
"aip1이라는 차량의" cmd_vel. **모든 차량이 동일한 토픽 세트**를 갖는다.

**3. 중앙 PC는 단일 장애점이 아니어야 한다(장기).** 각 차량에 cmd_vel freshness watchdog(0.5s)을 둔
이유. 장기적으로 중앙 PC는 "있으면 편하지만 없어도 운용 가능"해야 한다.

**4. 인터페이스는 사양에 무관하게 동일하다.** ESP32 보조 차량과 고사양 차량이 같은 토픽 스키마
(`FleetHeartbeat` 등)를 쓴다. 사양 차이는 내부 구현에서만 드러난다.

**5. Fleet Brain은 제안만 한다.** AI는 `/fleet/suggestions`만 발행하고, 운영자 승인 후에만 기존
명령 경로로 실행되도록 설계한다. 안전 정지(watchdog)는 AI와 분리된 반사 영역으로 유지.

## 6. 임시 결정 목록 (사양 향상 시 수정 예상)

| 파일 | 임시 결정 | 제거 조건 |
|---|---|---|
| `supervisor_node.py` / `supervisor.yaml` | vehicle_ids 정적 설정 | 차량 자동 발견으로 교체 |
| `coordinator_node.py` | 중앙 PC에 위치 | 각 차량 탑재 시 이동 |
| `scout_localizer_node.py` | 메인 카메라 기반 측위 | 차량별 독립 측위 확보 시 선택 사용 |
| `central.launch.py` / `docker-compose.yml` | 중앙 PC 전체 실행 | 각 차량 launch/컨테이너로 분산 |
